  • 2. HTML Language Basics

    Here are a couple of key elements that must be well understood to have a good grasp of HTML Programming Language Basics

  • A. The Architecture of the Web

    The web is a bunch of computers that communicate with each other. When a person goes to a web page like hillaryclinton.com or donaldjtrump.com, their computer sends a HTTP Request to a server. The server finds the appropriate HTML document and sends it back to the user's computer where a web browser (e.g. IE, Chrome, Firefox) interprets the page and displays it on the user's screen.

  • B. How HTML works

    HTML stands for Hypertext Markup Language. HTML documents form the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks". HTML documents can also contain references to other content like videos etc.

  • C. Elements of HTML

    In this section we covered Tags & Elements, Inline vs. Elements, plus HTML Document Structure. HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. This online resource explains the various flavours of tags within the HTML ecosystem. HTML documents follow a specific structure.

  • D. DOM Strucutre

    We then talked about these in the context of web development, examining the role of html, css and javascript and the key foundational elements of front end design providing structure, style and interactive elements respectively. We then talked about DOM, the Document Object Model, structure within a webpage and using the div tag to create boxes that form the basis of the layout of content within the page. We learned how class attributes can be compared to labels on these boxes

  • E. CSS Styling

    This is a really broad topic but the foundational elements are all about creating boxes within your webpage and then using CSS, a Cascading Style Sheet as a central reference to apply features like font colour etc. to your basic html content. This online resource for the basic syntax of a CSS rule; lists all standard CSS properties, pseudo-classes and pseudo-elements.

  • F. Layout & Boxes

    We saw how to link the class objects defined in the html to the corresponding style elements in css. We learned a lot about the importance of boxes in front end web development including terms like boxifying, positioning of boxes (including flexbox layout) and ways to fine tune layout using the developer tools functionality.
